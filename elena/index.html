CSS: enough is enough

Big projects come with even larger CSS codebase. It’s too big and rather unpredictable very often. The problem originates from several developers working independently or too tough deadlines or lack of code architecture or all these at the same time. The result is always devastating — we need to face it — while CSS is considered to be “simple” language its power is tremendous and its impact to project’s life is great.

Your team may have everything working — sparkingly fast back-end, a mature front-end — but let you have a tiny interface element broken and all these brilliantly-developed stuff will make no sense at all.

So yes, you got it right — you should be able to maintain, extend, scale and reuse your CSS easily. Mind the first idea: enough is enough, you should have exactly as much code as you need to understand it, yet as little as possible.
